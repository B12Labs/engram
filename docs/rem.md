# REM — Rapid Eye Movement markdown dialect

> **REM = Rapid Eye Movement.** The portable markdown dialect every `.egm` bundle exports to. Named after REM sleep — when biological brains replay memories as vivid, readable narratives. REM is what a human *reads* when they `git clone` their memory.

## Purpose

`.egm` bundles are binary — fast for LEANN + Tantivy + PageIndex queries, opaque to humans. REM is the companion markdown mirror every bundle exports to.

- Read path: REM is a snapshot. Queries go through `.egm`, not REM.
- Write path: UI writes to `.egm` only. REM is generated.
- Edit path: user-edited `.rem.md` flows back into `.egm` on the next Dream Cycle, tagged `@provenance edited-by=user`.
- Git sync: `engram export --git` pushes REM into the user's private repo. SHA-256 idempotency means re-syncs are no-ops.

## Dialect extensions

REM extends GitHub Flavored Markdown with four comment-scoped constructs:

```markdown
<!-- @entity:person:ava-sterling -->
Ava met with the board on Tuesday...

<!-- @link:meet.egm#morn-standup-042 -->

<!-- @provenance source=meet edited-by=user confidence=0.92 -->

<!-- @embed:image r2://boss-media/u42/sunset-042.webp -->
```

- `@entity` — canonical entity reference (person, brand, topic, project)
- `@link` — cross-bundle reference to another `.egm` record
- `@provenance` — source / confidence / edit-history tags
- `@embed` — asset reference (image, audio, video) by R2 URI

Using HTML comments keeps REM 100% renderable in any Markdown viewer (GitHub, Obsidian, VSCode preview) — the tags degrade gracefully to invisible metadata.

## Folder layout

```
{userId}/
  unified.egm                  ← canonical binary bundle
  unified.rem.md               ← REM markdown mirror (generated by Dream Cycles)
  meet.egm + meet.rem.md
  .rem/
    index.md                   ← table of contents
    entities/                  ← one .md per canonical entity
    timeline.md                ← chronological log
```

## Why REM and not "just markdown"

Plain markdown loses the edges of the entity graph. REM preserves them via `@entity` + `@link` tags, so a round-trip through Git doesn't collapse the structure that LEANN + PageIndex depend on.

Same pattern as iCloud Photos — the originals live in the cloud, `~/Pictures` is a materialized view. `.egm` is canonical; REM is the materialized view.
